# QA Playbook：v2.0 Android App UI 1:1 重写验收

> **对应 plan**：`docs/superpowers/plans/2026-05-12-android-app-ui-redesign-hybrid.md`
> **设计基准**：`ui/app-agent.html`（9 屏 phone-screen 设计稿，唯一 source of truth）
> **架构**：混合方案 — WebView 5 屏 + Compose 原生 4 屏
> **预计工时**：60-90 min（含 Part A 自动化首跑 + Part B 手测）

---

## 总览

本次重写把 Android App UI 与 `ui/app-agent.html` 设计稿做 1:1 对齐。验收分两部分：

| Part | 屏数 | 屏 | 验收方式 | 通过阈值 |
|------|------|----|----------|----------|
| **A — WebView 自动化** | 5 | S1 工作台 / S5 案件列表 / S6 案件详情 / S7 通话记录 / S8 个人信息 | Playwright 像素 diff | 单屏 ≤ 5% diff + 关键交互 OK |
| **B — Compose 手测** | 4 | S2 拨打请求 / S3 通话中 / S4 通话结束 / S9 强制退出 | adb 触发 + 肉眼对照 mockup 截图 | 视觉关键点齐全 + 关键交互 OK |

**为什么 Compose 不进自动化**：Playwright 跑 Chromium，无法测原生 UI。Layout Inspector + Emulator 截图比强行接 Espresso 截图测试更省时。

## Part A — WebView 5 屏（自动化像素 diff）

> Playwright 把 React 移动屏渲染到 390×844 viewport 截图，与 `ui/app-agent.html` 设计稿截图做 pixelmatch 对比。
> 阈值 5%（spec 原意 2%，但 Refine + react 18 + tailwind 渲染 vs 静态 HTML 字体 / 图标存在不可避免漂移；放宽到 5% 才能 actionable）。

### A.1 第一次跑：生成 baseline

```bash
cd frontend
npm run visual:baseline
```

产物落在 `frontend/tests/visual-baselines/v2.0-mobile/`，每张约 50-80 KB；**必须 commit 进 git**（这是设计稿的版本化截图）。

### A.2 后续跑：diff

```bash
# 后端 + frontend dev server 已起
cd frontend
npm run visual:diff
```

输出：
- 实际截图 → `frontend/tests/visual-actuals/v2.0-mobile/{screen}.png`
- 差异图 → `frontend/tests/visual-diffs/v2.0-mobile/{screen}.diff.png`
- 控制台打印 `[s1-home] diff = 3.45% (12423/360120 px) → ...diff.png`

actual + diff 已加进 `frontend/.gitignore`，每次跑都重新覆盖。

### A.3 5 屏明细

> 每屏验收 = （像素 diff ≤ 5%）+ （交互 checklist 全过）。
> 如果 diff 失败但布局正确（仅字体 / icon 微差），先看 diff PNG 评估：
> - 红色密集分布在文字 → 字体 hinting 差异，可接受，记录到「已知差异」
> - 红色集中在某区域 → 真实 layout 漂移，必须修

### B.1 S2 拨打请求 — `DialRequestActivity`

源码：`poc/android/app/src/main/java/com/autoluyin/demo/screens/dial/DialRequestActivity.kt`
设计稿 section：`ui/app-agent.html#app-dial-request`

#### 触发方式

```bash
adb shell am start -n com.autoluyin.demo/.screens.dial.DialRequestActivity \
  --el call_id 12345 \
  --el case_id 67890 \
  --es owner_name "张大伟" \
  --es owner_phone "13800138000" \
  --es owner_phone_masked "138****8000" \
  --es building "3栋1单元" \
  --es room "1201室" \
  --es amount_owed "¥8,420" \
  --ei months_overdue 7 \
  --es last_contact_at "3天前" \
  --es last_outcome "推托"
```

> 注：上面 `--el` 是 long extra，`--es` 是 string extra，`--ei` 是 int extra。
> `expires_at` 不传时 Activity 默认 +15s 倒计时（`DEFAULT_TTL_MS`）。

#### 视觉对照 checklist

- [ ] 蓝色垂直渐变背景 `#1E3A5F → #1A56DB`（status bar 也染色 — `WindowCompat.setDecorFitsSystemWindows(window, false)`）
- [ ] 顶栏：`📲 主管发来外呼请求` 14sp 白半透 + 实时时钟（HH:mm）右对齐
- [ ] 80×80 圆形头像：半透明白底，业主姓首字（`avatarChar`）32sp 白居中
- [ ] 业主名 24sp bold 白 + pill 风格房号（`roomLabel` = `3栋1单元 1201室`）
- [ ] 半透明白卡内 3 行：
  - 金额行：32sp 黄色 `#FBBF24`「¥8,420」+ 灰字 `欠缴 7 个月`
  - 上次联系：14sp 白半透 `上次联系：3天前 · 推托`
- [ ] 110×110 倒计时环：白色 stroke 6dp，从 12 点位置顺时针缩进
- [ ] 倒计时数字 28sp 白居中 + 「N 秒后自动取消」副文案
- [ ] 双按钮：
  - 绿色「立即拨打」满宽 56dp，绿底 `#10B981` 白字
  - 透明白边「稍后处理」满宽 56dp 底部

#### 交互 checklist

- [ ] 倒计时 15 → 0 自动 finish + logcat `auto-cancelled (countdown expired)`
- [ ] 点「立即拨打」→ ACTION_CALL 触发系统拨号（号码 = `phoneToDial`，明文优先）→ Activity finish
- [ ] 点「稍后处理」→ Activity finish + logcat `user deferred`
- [ ] 锁屏状态触发该 Activity → 屏幕被点亮 + 解锁键盘弹出请求（API 26+）
- [ ] 后退键被屏蔽（防误退到不存在的上一屏）

#### 已知差异

| # | 项 | 影响 | 责任 |
|---|---|---|---|
| 1 | 设计稿用 lucide phone icon，实现用 Material Icons Phone | 视觉接近，可接受 | — |
| 2 | `owner_phone` 当前在 push payload 内明文走 MiPush | 合规风险 | 后端补 dial-info token 模式 |
| 3 | `phoneToDial` 拿到打码号时拨号会失败（系统识别不出 `*`） | 用户感知 | 同上修复 |
| 4 | defer 没调后端 endpoint，仅 `Log.i` | 主管端看不到 deferred 状态 | Task v2.1 后端补 endpoint |

#### 触发方式

```bash
adb shell am start -n com.autoluyin.demo/.screens.auth.ForceLogoutActivity \
  --es code "ERR_SESSION_EVICTED" \
  --es message "您的账号于 14:35 在 Android 设备（小米12）上登录"
```

实际触发场景：
- 同账号在另一台设备登录 → 后端踢旧 session → SDK 收到 401 + `ERR_SESSION_EVICTED` → 走 `ForceLogoutActivity.startFor(reason)`

- [ ] 80×80 红圆图标：

  - 锁 emoji 🔒 32sp 居中
- [ ] 标题 18sp bold `账号已在其他设备登录`
- [ ] 描述 13.5sp lineHeight 1.7 `您的账号于 14:35 在 Android 设备（小米12）上登录`
- [ ] 灰色提示卡 `如非您本人操作，请立即修改密码并联系管理员`
- [ ] 蓝色「重新登录」按钮满宽 56dp
- [ ] 蓝色下划线「联系管理员」link

#### 交互 checklist

- [ ] 返回键被屏蔽（防退回到鉴权失败的 Activity，触发死循环）
- [ ] 点「重新登录」→ 跳 MainActivity → preflight 自动弹登录 dialog
- [ ] 点「联系管理员」→ toast 占位（PoC 阶段）
