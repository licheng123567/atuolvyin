# QA Playbook：法务转化「内部处理」环节（v1.9.0 方案 B）

> **目的**：v1.9.0 把法务转化流程从「supervisor 审批 → admin 撮合律所」改为「supervisor 审批 → **物业法务内部处理** → 关闭 / 升级律所」。本 playbook 验证内部处理环节全流程。
> **预计工时**：20-30 min
> **前置**：v1.8 playbook 阶段 1-3 已通过（agent 申请 + supervisor 审批通过）

---

## 阶段 3-config — admin 一次性后台配置（5 min）

> 这是 v1.9.0 新增管理后台，第一次跑必须先建数据，否则法务老周「出具律师函」无可选模板/律所。

**B 窗口登录 `13000000002` / `Demo@123!`**

### 3a. 新增合作律所
1. 左侧 **`合作律所`** → 「+ 新增律所」
2. 填：
   - 名称：`Demo 法务公司`（与平台 LawFirm 重名 OK，本表独立）
   - 联系人：`王律师`
   - 联系电话：`13800001111`
   - 邮箱：`wang@demolaw.cn`
   - 备注：`长期合作；律师函按 200 元/份`
3. 保存 → 列表出现一行
4. **【验收点 W1.1】** 联系电话以脱敏形式显示（`138****1111`）

### 3b. 新增律师函模板
1. 左侧 **`律师函模板`** → 「+ 新增模板」
2. 模板已预填：
   - 名称：`物业费追缴律师函（标准版）`
   - 类别：律师函
   - 正文：含 `{{owner_name}}`、`{{building}}`、`{{room}}`、`{{amount_owed}}` 等占位符
   - 变量列表：8 行
3. 保存 → 列表出现 1 条
4. **同样建一份催告函**（类别选「催告函」，正文删掉律师签字行）

## 阶段 4 — 法务老周「待内部处理」工作台（5 min）

**A 窗口登录 `13000000006` / `Demo@123!`**

1. 左侧 **`待内部处理`** 菜单 → 列表里看到上轮 supervisor 审批通过的订单
2. **【验收点 W2.1】** tabs 区分「待处理 / 已关闭 / 全部」，默认「待处理」展示 status=`internal_processing` 的订单
3. **【验收点 W2.2】** 列表显示：业主姓名+脱敏电话、房号、欠费金额、欠费月数、状态 badge、申请人、处理次数（初始 0）
4. 点 **「处理」** → 进详情页（业主信息 + 处理记录时间线 + 右栏操作按钮组）

---

## 阶段 5 — 记录沟通 + 出具律师函（10 min）

**仍在详情页**

### 5a. 记录电话沟通
1. 右栏 **「记录沟通」** → 表单：
   - 备注：`已联系业主，业主称物业服务质量有异议，要求调解后再缴`
2. **「保存记录」**
3. **【验收点 W3.1】** 中栏处理记录时间线新增一条 📞 节点，文案显示备注

### 5b. 出具律师函
1. 右栏 **「出具律师函」** → 表单：
   - 模板下拉：选「物业费追缴律师函（标准版）」
   - 合作律所下拉：选「Demo 法务公司」
   - 备注：`首次正式发函，要求 7 日内回款`
2. **「保存记录」**
3. **【验收点 W3.2】** 中栏新增 📨 节点，显示「模板：物业费追缴律师函 · 律所：Demo 法务公司」
4. **【验收点 W3.3】** 案件详情页（B 窗口 admin 进同案件）也能看到这条 📨 节点（v1.9.0 时间线集成）

### 5c. 记录调解
1. 右栏 **「记录调解」** → 备注：`物业减免 20% 滞纳金 + 业主分期 6 个月，达成共识`
2. 保存
3. **【验收点 W3.4】** 中栏新增 🤝 节点

---

## 阶段 6 — 关闭订单（已承诺）（3 min）

**仍在详情页**

1. 右栏底部「关闭订单」区 → 点 **「📝 已承诺」**
2. 弹关闭表单：备注 `按调解协议执行；首期 5/15 缴费`
3. **「确认关闭」**
4. **【验收点 W4.1】** 订单 status 变 `closed_promised`
5. **【验收点 W4.2】** 右栏从「操作按钮组」变为「订单已关闭 · 关闭原因：达成承诺 + 关闭时间」
6. **【验收点 W4.3】** 中栏处理记录最末新增 ⚖️/⚖ 节点（关闭事件）

---

## 阶段 7 — 闭环验证（admin 视角）（3 min）

**B 窗口 13000000002**

1. 进入对应案件详情页 → 中栏活动时间线应有完整链条：
   - 案件创建 → 通话 → 申请转法务 → 审批通过
   - 📞 法务沟通业主
   - 📨 出具律师函（含模板名 + 律所名）
   - 🤝 调解会议
   - ⚖️ 法务订单关闭 · 达成承诺
2. **【验收点 W5.1】** 案件 stage 自动 → `promised`（v1.9.0 close_promised 联动）
3. **【验收点 W5.2】** 业主画像 3 统计卡：「联系次数」「承诺次数」「工单数量」（v1.8.0）
4. **【验收点 W5.3】** 法务老周「待内部处理」列表的「待处理」tab 该订单消失，「已关闭」tab 出现

## 阶段 8 — 升级律所埋点（2 min）

**A 窗口法务老周，新选另一个未关闭的内部订单**

1. 右栏底部 **「⚠️ 升级到律所」** → 确认弹窗 → 确认
2. **【验收点 W6.1】** 订单 status=`escalated_to_lawfirm`
3. **【验收点 W6.2】** 案件时间线新增 ⚠️ 升级节点
4. **【验收点 W6.3】** 案件 stage **不变**（升级到律所不联动 paid/promised/closed）
5. **方案 C 范围**：admin 后续在「法务转化」inbox 看到这条 escalated 订单 → 撮合具体律所；本轮先留埋点

## 🟡 P1 顺手测的

- **多 membership**：13000000006 法务老周右上角下拉，应只有 1 个 `legal` membership（律所内身份不在租户切换器里）
- **权限**：13000000004 内勤小张访问 `/legal/internal-orders` 应 403（仅 legal/admin/supervisor 可访问）
- **Admin 后台权限**：13000000003 督导访问 `/admin/partner-law-firms` 应只读（GET 返 200，POST 返 403）
- **关闭后保护**：法务老周对已关闭订单 `POST /actions` 应 409 ERR_ORDER_CLOSED

## 🐛 常见问题

| 现象 | 排查方向 |
|---|---|
| 「出具律师函」表单模板下拉为空 | 先到 admin → 律师函模板 新建至少 1 条 |
| 「合作律所」下拉为空 | 同上，admin → 合作律所 |
| 时间线节点不显示 | 检查后端 `case_timeline.py` 是否聚合了 `LegalInternalAction`；前端 `ActivityTimeline.eventMeta` 是否覆盖 7 种 type |
| 关闭订单后 case.stage 不变 | 检查 close endpoint 的 `_legal_stage_for_close_reason` 映射 |
