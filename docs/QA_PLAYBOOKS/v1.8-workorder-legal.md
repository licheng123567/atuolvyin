# QA Playbook：工单 + 法务全链路（v1.8.0）

> **目的**：v1.8.0 完工后的端到端业务闭环手测脚本。一次走通可清掉 90% 业务风险。
> **预计工时**：30 min
> **覆盖**：catch-up agent → supervisor 审批 → admin 撮合 → 协调员处理工单 → 法务建案出文书

## 阶段 1 — 催收员发起两个流程（13000000004 内勤小张）

**A 窗口登录 `13000000004` / `Demo@123!`**

1. 进入工作台：左侧 `工作台` → 选择 col-1 第一个案件
2. **【验收点 P1.1】** col-2 业主画像：3 统计卡片 + 欠款月份 + 最近通话记录 accordion（v1.8.0 改造）
3. 点击右栏 **「申请减免」** 按钮 → 弹 `DiscountRequestModal`：
   - 减免金额填 `2000`（自动判定走督导审批）
   - 减免原因选「经济困难」
   - 备注：`业主父亲住院，月底承诺一次性结清`
   - 点 **「提交申请」** → toast「减免申请 #X 已提交」
4. 同案件，点击右栏 **「申请转法务」**：
   - 弹 prompt 输入转法务理由
   - 填 `已多次失联，建议升级法务` → 确认
   - toast「申请转法务 已记录」
5. **【验收点 P1.2】** 切回案件详情，活动时间线应新增 2 条系统事件

## 阶段 2 — 督导审批两个申请（13000000003 督导小李）

**B 窗口登录 `13000000003` / `Demo@123!`**

1. 左侧 **`减免审批`** → 看到 `2000` 元减免：

## 阶段 4 — admin 派工单给协调员（13000000002）

仍在 B 窗口（admin）

1. 进入案件详情页 → 右栏 **「创建工单」**
2. 填工单内容：`核实业主父亲住院凭证，下周三前回访确认是否一次性结清` → 确认
3. **【验收点 P4.1】** toast「工单 #X 已创建」+ 时间线新增「工单创建」节点
4. 左侧 **`工单看板`** → 工单 status=`open`，未分配
5. 点 **「分配协调员」** → 选 **`协调员小赵`** → status=`assigned`

## 阶段 5 — 协调员处理工单（13000000007 协调员小赵）

**A 窗口（先登出小张）登录 `13000000007` / `Demo@123!`**

## 阶段 6 — 法务老周建案 + 出文书（13000000006）

**B 窗口（先登出 admin）登录 `13000000006` / `Demo@123!`**

1. 左侧 **`法务案件`** → 列表里看到刚撮合的案件
2. **【验收点 P6.1】** v1.7.0 电话可见性：案件 stage=`pending_eval` 属 active 阶段，**业主电话应明文**
3. 点详情 → 看到完整案件信息 + 通话记录 + 工单
4. 点 **「创建法务案件」** → stage 推进到 `evidence_collection`
5. 进入「证据」tab → 上传一份测试 PDF → 验收上传成功
6. 进入「法律文书」tab → 选模板「催款律师函」→ 选模板变量 → 出具 → 下载预览（PDF）
7. **【验收点 P6.2】** 文书生成后可下载，案件时间线新增「法务文书已出具」

---

## 阶段 7 — 闭环验证（13000000002 admin）

**B 窗口切回 admin**

1. 进入同一案件详情 → 时间线应有完整链条（自上而下）：
   - 案件创建（导入）
   - 通话 N 通
   - 转化为法务（pending → approved → assigned → in_progress）
   - 减免审批通过
   - 工单创建 / 完成
   - 法务案件 stage 变更
   - 法务文书已出具
2. **【验收点 P7.1】** 业主画像 3 统计卡片：工单数量 ≥ 1
3. **【验收点 P7.2】** 右栏「转交法务处理」按钮应禁用或显示「已转法务」

## 🟡 P1 顺手测的（30 min 内可加）

- **多 membership 切换**：13000000003（督导）右上角下拉切「催收员（内勤）」身份 → 工作台看分配给自己的案件
- **电话脱敏翻转**：B 窗口 13000000005 → 看一个案件电话明文 → DB 跑 `UPDATE provider_tenant_contract SET status='terminated' WHERE provider_id=<X>` → 前端刷新 → 立即脱敏
- **「记录跟进」快捷入口**（v1.8.0）：admin/agent/supervisor 三个列表各点一次「📝 记录跟进」按钮 → Modal → 保存 → 列表刷新

## 🐛 如果某步失败

1. 后端日志：`tail -f poc/backend/uvicorn.log`（或控制台）
2. 前端 Network tab 看 API 响应
3. 复刻 bug 路径到 `docs/superpowers/plans/<日期>-bug-<标题>.md`
4. 单独提 PR fix；不要在大 PR #14 里塞 hotfix
