# 有证慧催 — 设计规范与交付测试标准

**版本**：v1.0  
**日期**：2026-04-28  
**适用范围**：PC 端（Refine.dev + shadcn/ui）+ Android App（Kotlin）

## 一、Design Token

### 1.1 色彩

| Token | 值 | 用途 |
|-------|----|------|
| `color-primary` | `#1A56DB`（蓝）| 主按钮、链接、激活状态 |
| `color-primary-hover` | `#1E429F` | 主按钮悬停 |
| `color-danger` | `#E02424`（红）| 风控告警 L2/L3、删除操作、超期欠款 |
| `color-warning` | `#D97706`（橙）| 风控 L1、到期预警、待处理状态 |
| `color-success` | `#057A55`（绿）| 已缴费、承诺已履行、操作成功 |
| `color-neutral-900` | `#111827` | 正文 |
| `color-neutral-600` | `#4B5563` | 次要文字、标签 |
| `color-neutral-200` | `#E5E7EB` | 分割线、边框 |
| `color-neutral-50`  | `#F9FAFB` | 页面背景 |
| `color-surface`     | `#FFFFFF` | 卡片、弹窗背景 |
| `color-agent-bubble`| `#DBEAFE` | 通话转写：催收员气泡（近场）|
| `color-owner-bubble`| `#F3F4F6` | 通话转写：业主气泡（远场）|

**禁止**：不得在界面中使用色彩传达唯一信息（需同时有图标或文字标注），满足无障碍 AA 对比度标准（≥4.5:1）。

### 1.2 字体

| 层级 | 规格 | 用途 |
|------|------|------|
| H1 | 24px / Semi-Bold | 页面主标题 |
| H2 | 18px / Semi-Bold | 区块标题、弹窗标题 |
| H3 | 15px / Medium | 卡片标题、表格列头 |
| Body | 14px / Regular | 正文、表格内容 |
| Small | 12px / Regular | 辅助说明、时间戳、徽章 |
| Code | 13px / Mono | 手机号（脱敏展示）、JSON |

App 端字体同上，最小可交互元素字号 ≥ 14px。

### 1.3 间距（8px 基准）

```
4px  — 图标与文字间距
8px  — 组件内间距（padding-xs）
16px — 卡片内边距（padding-md）
24px — 区块间距（padding-lg）
32px — 页面顶部留白
```

### 1.4 圆角与阴影

| 场景 | 值 |
|------|----|
| 按钮、输入框、徽章 | `radius: 6px` |
| 卡片、弹窗 | `radius: 8px` |
| 底部弹出面板（App）| `radius-top: 16px` |
| 卡片阴影 | `0 1px 3px rgba(0,0,0,0.10)` |
| 弹窗阴影 | `0 8px 24px rgba(0,0,0,0.15)` |

## 二、组件规范（shadcn/ui 约束）

### 2.1 组件优先级

> **必须**优先使用 shadcn/ui 内置组件，禁止重复造轮子。

| 场景 | 使用组件 |
|------|---------|
| 数据表格 | `<DataTable>` + Refine `useTable` |
| 表单 | `<Form>` + Refine `useForm` |
| 弹窗/确认框 | `<Dialog>` / `<AlertDialog>` |
| 下拉选择 | `<Select>` / `<Combobox>` |
| 日期选择 | `<DatePicker>` |
| 通知 | `<Toast>`（右下角，最多 3 条堆叠）|
| 状态徽章 | `<Badge>`（颜色映射见 1.1）|
| 骨架屏 | `<Skeleton>`（所有异步加载区域必须有）|
| 空状态 | 统一空状态组件：图标 + 主文案 + 可选操作按钮 |

### 2.2 按钮规则

| 类型 | 场景 | 样式 |
|------|------|------|
| Primary | 每页最多 1 个主操作 | 蓝色实心 |
| Secondary | 次要操作 | 白色描边 |
| Destructive | 停用/删除 | 红色，需二次确认 `<AlertDialog>` |
| Ghost | 表格行内操作 | 无背景，悬停变灰 |

**禁止**：表格行内出现 Primary 按钮；删除/停用操作必须有 AlertDialog 二次确认。

### 2.3 表单规则

- 必填项在 label 右侧标 `*`（红色）
- 校验时机：失焦即时校验 + 提交整体校验
- 错误提示紧贴输入框下方，颜色 `color-danger`，不用 Toast
- 手机号输入框：实时格式化为 `138 0000 0000`；后端存储去格式化原始字符

### 2.4 加载与错误状态

| 状态 | 处理方式 |
|------|---------|
| 列表加载 | 骨架屏（≥500ms 才显示，避免闪烁）|
| 提交中 | 按钮 loading 旋转 + 禁用，防止重复提交 |
| 请求失败 | Toast 错误通知 + 具体错误原因（非"请求失败"等泛词）|
| 空列表 | 空状态组件，区分"无数据"和"搜索无结果" |
| 网络断开 | 顶部全宽黄色 Banner："网络已断开，正在重连…" |

## 三、布局规范

### 3.1 PC 端整体结构

```
┌──────────────────────────────────────────────────────┐
│  TopBar：Logo + 当前租户名 + 角色标签 + 通知铃 + 头像  │ 56px
├──────────┬───────────────────────────────────────────┤
│          │                                           │
│  SideNav │  主内容区                                  │
│  240px   │  padding: 24px                            │
│  折叠→48px│  最大宽度: 1440px，居中                    │
│          │                                           │
└──────────┴───────────────────────────────────────────┘
```

- SideNav 菜单项按角色权限动态渲染（Refine `accessControlProvider`）
- 无权限菜单项直接隐藏（不显示禁用态）
- 页面标题 + 面包屑固定在内容区顶部

### 3.2 催收员实时通话工作台（三栏布局）

```
┌──────────┬──────────────────────┬────────────────────┐
│ 案件列表  │  实时转写对话流        │  AI 建议 + 快捷操作 │
│  280px   │  flex-grow           │  320px             │
│          │  气泡颜色区分说话人    │  卡片按优先级排序   │
└──────────┴──────────────────────┴────────────────────┘
```

- 右侧 AI 建议区：新卡片从顶部插入，最多展示 5 条，旧卡片淡出
- 风控告警 L2/L3 全屏遮罩（z-index 最高），需主动关闭

### 3.3 Android App 导航结构

```
底部 Tab（4 项）：工作台 / 案件 / 通话记录 / 我的
```

| Tab | 页面 |
|-----|------|
| 工作台 | 当日任务、等待拨打提示、AI 建议卡片 |
| 案件 | 私海案件列表、案件详情 |
| 通话记录 | 本人历史通话、分析结果查看 |
| 我的 | 账号信息、设置、退出登录 |

- 最小触摸目标：48×48dp
- 通话中状态：底部 Tab 替换为通话控制栏（挂断 / 静音 / 备注）
- 弹出通知（DIAL_REQUEST）：全屏接听卡片，2 个操作按钮（立即拨打 / 稍后）

### 3.4 响应式断点（PC 端）

| 断点 | 宽度 | 适配 |
|------|------|------|
| Desktop | ≥1280px | 完整三栏 / 侧边栏展开 |
| Tablet  | 768-1279px | 侧边栏默认折叠，双栏布局 |

> 注：平台运营后台（admin.youcuihuicui.com）**不需要**适配移动端。

## 四、数据展示规范

### 4.1 手机号脱敏

| 场景 | 展示格式 |
|------|---------|
| 列表（agent/supervisor 查看）| `138****8888` |
| 案件详情（admin 查看）| `138 0000 8888`（完整，有权限） |
| 转写文本内出现的手机号 | 自动正则替换为 `[手机号已隐藏]` |
| 支付 H5 页面 | 不展示任何手机号 |

### 4.2 金额与日期

- 金额：`¥1,234.00`（保留两位小数，千分位分隔符）
- 日期：`2026-04-28`（ISO 格式，不用"04/28/2026"）
- 相对时间：距今 ≤24h 显示"3小时前"，超过 24h 显示绝对日期
- 时长：`2:34`（分:秒），超过 1 小时显示 `1:02:34`

### 4.3 状态映射（Badge 颜色）

| 状态 | Badge 变体 | 文案 |
|------|-----------|------|
| 待联系 | neutral | 待联系 |
| 跟进中 | blue | 跟进中 |
| 承诺缴费 | warning | 承诺缴费 |
| 已缴费 | success | 已缴费 |
| 已升级 | purple | 已升级 |
| 已关闭 | neutral | 已关闭 |
| 风控 L1 | warning | 注意 |
| 风控 L2 | danger | 警告 |
| 风控 L3 | danger（实心）| 紧急 |

## 五、开发交付 Checklist

每个页面/功能合并前，开发者自检以下条目（均为 Must Pass）：

### 5.1 权限与多租户

- [ ] 所有接口携带 `tenant_id`，后端已校验不得跨租户读写
- [ ] 前端菜单/按钮按角色权限动态显示（Refine accessControlProvider 接入）
- [ ] 平台运营员接口已校验 `PlatformOpsAssignment`，不能操作未分配的租户/服务商
- [ ] 无权限路由访问时返回 403 页面（非空白页）

### 5.2 数据安全

- [ ] 手机号在 API 响应中已按权限脱敏（非 admin 角色返回 `138****8888`）
- [ ] 转写文本已过正则脱敏（手机号替换为 `[手机号已隐藏]`）
- [ ] 支付 H5 页面 URL 使用 token（非 case_id），token 单次有效，30 天过期

### 5.3 接口规范

- [ ] 列表接口支持分页（`page` + `page_size`，默认 20）、排序、筛选
- [ ] 接口响应格式统一：`{ code, message, data, pagination? }`
- [ ] 错误响应含可读 `message`（中文），不暴露堆栈信息
- [ ] 幂等操作（创建/支付）有防重机制（`idempotency_key` 或数据库唯一约束）

### 5.4 性能

- [ ] 列表页首屏数据加载 ≤ 2s（P90，生产环境）
- [ ] 骨架屏在数据加载期间正确显示
- [ ] 大列表（>100 条）使用虚拟滚动或服务端分页，禁止前端全量加载

### 5.5 前端质量

- [ ] TypeScript 无 `any` 类型（eslint `@typescript-eslint/no-explicit-any: error`）
- [ ] 表单提交中状态：按钮 loading，不可重复点击
- [ ] 删除/停用操作有 `<AlertDialog>` 二次确认
- [ ] 空状态、错误状态、加载状态均已实现（非空白区域）
- [ ] 手动测试：刷新页面不丢失当前状态（列表筛选条件存入 URL query）

### 5.6 Android App 专项

- [ ] 已在小米/华为/OPPO 真机各测一台（非模拟器）
- [ ] 厂商推送（小米/华为/OPPO Push SDK）在后台杀进程后仍能收到 DIAL_REQUEST
- [ ] 录音文件上传失败有重试逻辑（最多 3 次，指数退避）
- [ ] 网络切换（WiFi→4G）后 WebSocket 自动重连，不需要用户手动重启 App
- [ ] L3 风控自动挂断：默认关闭，管理员后台开启后方可触发

## 六、测试验收标准

### 6.1 功能测试（按角色）

每个角色上线前必须通过以下核心用例：

| 角色 | 必测用例 |
|------|---------|
| platform_superadmin | 创建运营员账号并分配租户；全局风控词新增后在对应租户通话中触发 |
| platform_ops | 只能看到自己分配的租户/服务商；开通试用账号全流程；短信通知发出 |
| admin | 导入 100 条业主名单无报错；开通/停用员工账号；查看全公司看板数据 |
| supervisor | 复核通话并修正说话人标注；强制转接（SUPERVISOR_TAKEOVER）流程 |
| agent_internal | App 接收 DIAL_REQUEST → 拨打 → 通话中看到 AI 建议 → 结束通话后案件状态更新 |
| agent_external | 仅能使用 App；私海案件查看；不能访问 PC 端任何页面 |
| legal | 接收法务升级案件；上传诉讼文档；更新案件进展 |
| workorder | 查看/处理工单；不能看到手机号和欠费信息 |
| provider_admin | 查看应收账款；提交结算异议；跨租户数据不可见 |

### 6.2 性能基准

| 指标 | 目标值 | 测试方法 |
|------|--------|---------|
| 实时 ASR 首字延迟 | ≤ 3s | 测试 30 通通话，取 P90 |
| AI 建议推送延迟（ASR→LLM→WebSocket） | ≤ 5s | 同上 |
| Aho-Corasick 风控关键词匹配 | ≤ 50ms | 单元测试，1000 词库 |
| 批量上传 500 文件（各 5MB）完成时间 | ≤ 30min | 并发 5 线程，实际环境 |
| 列表页首屏（P90） | ≤ 2s | Lighthouse / k6 压测 |
| WebSocket 重连成功率 | ≥ 99% | 模拟断网 100 次 |
| 单设备踢出到收到 FORCE_LOGOUT | ≤ 3s | 手动测试 |

### 6.3 安全测试（上线前必做）

- [ ] SQL 注入：所有输入参数使用参数化查询，Burp Suite 扫描无高危
- [ ] 越权访问：用 agent 账号的 token 请求 admin 接口，返回 403
- [ ] 跨租户：用租户 A 的 token 请求租户 B 的 case_id，返回 404（不暴露存在性）
- [ ] 暴力破解：连续 5 次错误密码后账号锁定，锁定状态下继续请求返回 423
- [ ] JWT 过期：使用过期 token 请求，返回 401；前端自动跳转登录页
- [ ] 文件上传：上传非音频文件（如 `.exe` 伪装为 `.mp3`），后端 MIME 验证拒绝
- [ ] XSS：话术内容、备注字段渲染时已转义，不执行脚本

### 6.4 合规验收

- [ ] 通话开始前 App 已录音授权提示（首次使用弹权限申请）
- [ ] 联系频次控制：同一业主 2 次/天上限在 CRM 层强制拦截，不依赖员工自觉
- [ ] 数据保留策略配置页可读（具体删除执行 v2.0，但字段和入口 MVP 已有）
- [ ] 平台审计日志不可删除（接口无 DELETE 操作，后台 DB 表无删除权限）

### 6.5 UAT 验收流程

```
1. 开发自测（上述 Checklist 100% 通过）
2. 测试环境部署 → QA 执行功能用例（覆盖率 ≥ 85%）
3. 性能压测通过基准值
4. 安全扫描无高/中危漏洞
5. 产品经理操作验收（模拟真实业务场景走一遍）
6. 合规条目确认
7. 正式环境灰度（10% 流量）→ 48h 无报警 → 全量
```

## 七、设计交付规范

### 7.1 设计师交付物

| 文件 | 格式 | 内容 |
|------|------|------|
| 组件库 | Figma（本地组件库）| 基于 shadcn/ui Token，含 5 种状态（default/hover/active/disabled/error）|
| 页面设计稿 | Figma | 按 PRD 第 21 节角色分 Frame，含空状态/错误状态/加载状态 |
| 标注说明 | Figma Dev Mode | 间距/颜色 Token 名称（非 HEX 值）、交互备注 |
| 图标 | SVG | 来自 lucide-react（与 shadcn/ui 同源），禁止混用其他图标库 |
| App 设计稿 | Figma | 1x/2x/3x，标注 dp（非 px） |

### 7.2 设计评审门禁

设计稿进入开发前，产品经理确认：

- [ ] 所有角色页面（78 个）均已覆盖，无遗漏
- [ ] 手机号脱敏展示符合第四节规范
- [ ] 无权限页面有明确的 403/空状态设计
- [ ] 关键破坏性操作（停用/删除）有二次确认弹窗设计
- [ ] 风控告警三级（L1/L2/L3）视觉层级明显可区分
- [ ] Android App 在 5.5 寸屏（1080×1920）上无内容溢出
